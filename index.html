Platform Intelligence

How the AI Backend Works

Anthropic Claude API

Every message is sent to Claude via the Anthropic /v1/messages API. The model receives your business profile as system context, ensuring responses are tailored to your exact statutory obligations.

Business Context Injection

The system prompt embeds your live business data employee count, VAT status, active obligations, overdue items and risk scores so Claude gives precise, context-aware compliance guidance, not generic answers.

Conversation Memory

Full conversation history is sent with each request. Claude remembers what you've discussed, enabling follow-up questions and multi-turn compliance analysis across complex obligation chains.

TECHNICAL ARCHITECTURE LIVE AI BACKEND
1
User submits question
Your message is captured from the textarea. The full conversation array (role: user / role: assistant pairs) is assembled with the new message appended.
2
Business context system prompt constructed
A system prompt containing your business profile company name, VAT status, employee count, active obligations, overdue items, and current risk score is assembled dynamically before each API call.
3
POST to Anthropic /v1/messages
A fetch request is sent with model: "claude-sonnet-4-20250514", the system prompt, and the full message array. Max tokens set to 1000 for detailed compliance responses.
4
Claude processes & responds
Claude analyses your question against the injected business context and UK compliance knowledge, returning a structured, precise answer referencing your actual obligation state.
5
Response rendered in chat UI
The response content is extracted, formatted with markdown-style rendering, and appended to the conversation view. The exchange is stored in the conversationHistory array for continuity.
